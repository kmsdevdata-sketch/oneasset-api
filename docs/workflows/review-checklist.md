# Review Checklist

## 리뷰 요청 전

- [ ] 변경 목적을 한 문장으로 설명할 수 있다.
- [ ] 관련 문서를 읽었고, 필요한 문서 갱신을 했다.
- [ ] 도메인 모델과 DB 모델의 차이를 설명할 수 있다.
- [ ] API 계약과 실제 DTO가 맞다.
- [ ] 예외/에러 응답 정책을 확인했다.
- [ ] `./gradlew compileJava` 또는 해당 단계의 최소 검증을 실행했다.
- [ ] 실행하지 못한 검증이 있으면 이유를 적었다.
- [ ] `git status --short`에서 의도하지 않은 staged/unstaged 변경이 없는지 확인했다.

## 리뷰할 때 보는 순서

1. 변경 의도와 DEV_LOG
2. 관련 문서
3. 도메인 모델
4. Application 유스케이스
5. Adapter와 외부 시스템 경계
6. 테스트와 검증 결과

## 리뷰에서 남길 내용

- 코드 결함
- 문서 불일치
- 보류된 설계 질문
- 다음 작업 후보
