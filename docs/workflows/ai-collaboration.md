# AI Collaboration Workflow

## 역할

AI를 생산성 도구로 쓰되, 설계 판단과 학습 흐름을 남기기 위한 협업 규칙을 기록한다.

## 기본 흐름

1. 사람이 먼저 작업한다.
2. 작업 중 판단과 막힌 지점을 `DEV_LOG.md`에 적는다.
3. AI에게 확인을 요청할 때는 변경 의도와 확인 범위를 말한다.
4. AI는 코드, 문서, 로그를 함께 보고 리뷰한다.
5. 확정된 정책과 구조는 팀 문서로 옮긴다.

## 확인 요청 문구 기준

| 요청 | AI가 해야 할 일 |
| --- | --- |
| `확인해봐` | 현재 변경분, DEV_LOG, 관련 문서를 읽고 문제/위험/다음 작업을 말한다. |
| `수정하지 말고 확인` | 읽기 전용으로 검토하고 멈춘다. |
| `완성해봐` | 기존 판단을 존중하면서 코드와 문서를 직접 패치하고 검증한다. |
| `막혔어` | 원인 후보를 좁히고 필요한 경우 공식 문서나 코드로 확인한다. |

## DEV_LOG에 쓰는 것

- 오늘 한 작업
- 왜 그렇게 판단했는지
- 아직 모르는 점
- AI에게 확인받고 싶은 지점

## 팀 문서로 승격하는 것

- 반복해서 참조할 정책
- 팀원이 알아야 하는 도메인 규칙
- API/DB 계약
- 되돌리기 어려운 아키텍처 결정

## 주의

- DEV_LOG는 개인 작업 로그라서 Git에 포함하지 않는다.
- accepted ADR이나 팀 정책 문서는 현재 합의된 사실만 담는다.
