# OneAsset Docs

이 디렉터리는 OneAsset의 팀 공용 기술 문서 공간이다.

문서는 코드와 함께 유지한다. 기능을 구현할 때는 관련 문서를 먼저 확인하고, 구현 후 실제 코드와 달라진 계약, 정책, 경계, 도메인 모델을 갱신한다.

## 문서 계층

| 경로 | 역할 | 언제 읽는가 | 언제 업데이트하는가 |
| --- | --- | --- | --- |
| `architecture/` | 시스템 구조, 계층 경계, 런타임/배포 관점 | 새 기능이 어떤 레이어에 들어갈지 판단할 때 | 레이어 경계, 패키지 구조, 외부 시스템 연동 방식이 바뀔 때 |
| `domains/` | 도메인 모델, DB 엔티티, 값 객체, 상태 전이 | 도메인 속성이나 영속성 모델을 만들기 전 | 도메인 필드, DB 컬럼, 관계, 상태 정책이 바뀔 때 |
| `policies/` | 인증, Lazy Sync, API Key, 예외, 시간, 삭제 같은 정책 | 코드에서 판단 분기가 생길 때 | 정책적 선택이나 예외 규칙이 추가될 때 |
| `api/` | 외부 API 계약과 DTO | 컨트롤러/응답 DTO를 만들기 전 | API path, request, response, error code가 바뀔 때 |
| `data/` | DB 스키마, 마이그레이션, 인덱스, 제약 조건 | 마이그레이션 작성 전 | 테이블, 컬럼, 제약, 인덱스가 바뀔 때 |
| `decisions/` | ADR: 되돌리기 어려운 기술 결정의 이유와 결과 | 큰 설계 결정을 새로 하거나 검토할 때 | 결정이 제안/승인/대체될 때 |
| `workflows/` | 작업 전후 체크리스트와 리뷰 절차 | 작업 시작 전, PR/커밋 전 | 팀 작업 방식이나 검증 절차가 바뀔 때 |
| `QUALITY_TOOLS.md` | 품질 도구 실행 방법 | 빌드/검증할 때 | 품질 도구나 명령이 바뀔 때 |
| `DEV_LOG.md` | 개인/AI 협업용 로컬 작업 로그 | 작업 중 사고 과정을 남길 때 | 작업 단위마다 필요 시 갱신한다. Git에는 포함하지 않는다. |

## 작업 전 확인 순서

1. `workflows/change-checklist.md`
2. 관련 `architecture/` 문서
3. 관련 `domains/` 문서
4. 관련 `policies/` 문서
5. 관련 `api/` 또는 `data/` 문서
6. 필요한 경우 `decisions/`의 ADR

## 작업 후 갱신 순서

1. 코드가 실제로 바꾼 계약과 정책을 확인한다.
2. `domains/`, `api/`, `data/`, `policies/` 중 영향을 받은 문서를 갱신한다.
3. 아키텍처 경계나 되돌리기 어려운 결정이면 `decisions/`에 ADR을 추가하거나 기존 ADR을 대체한다.
4. 검증 명령과 남은 위험을 작업 로그나 PR 설명에 남긴다.

## 참고한 문서화 모델

- C4: 시스템을 context, container, component, code 수준으로 나눠 설명한다.
- arc42: 목표, 제약, 컨텍스트, 빌딩 블록, 런타임, 배포, 결정, 품질, 리스크를 분리한다.
- ADR/MADR: 중요한 결정은 context, options, outcome, consequences로 남긴다.
- AWS ADR Guidance: 승인된 ADR은 직접 수정하지 않고 새 ADR로 대체한다.
